# Commitr — CLAUDE.md

## What This App Does
Commitr is a personal time tracking SPA. Users create projects/topics, run configurable timers per project, and review time spent aggregated by day, week, month, or year. All data is persisted in Supabase. There is no backend — the React app talks directly to Supabase via the JS SDK.

## Tech Stack
- **React** + **TypeScript** (strict mode)
- **Vite** (bundler)
- **Supabase** (Postgres DB + auth + REST SDK)
- **Vitest** + **React Testing Library** (unit + component tests)
- **Tailwind CSS** (styling)
- **Doppler** (secrets injection at runtime)
- **Vercel** (hosting)

## Data Model
```sql
projects (
  id           uuid primary key,
  name         text not null,
  color        text,
  created_at   timestamptz default now()
)

sessions (
  id               uuid primary key,
  project_id       uuid references projects(id) on delete cascade,
  started_at       timestamptz not null,
  ended_at         timestamptz,
  duration_seconds int generated always as
                   (extract(epoch from (ended_at - started_at))) stored
)
```

## Architecture

### Folder Structure
```
src/
  types/         ← TypeScript interfaces and enums, no logic
  components/    ← UI only, as dumb as possible, no business logic
  services/      ← API calls (Supabase) + business logic + data transformation
  hooks/         ← stateful React logic, composing services, UI interactions
  context/       ← global state (session, current user, active timer)
  lib/           ← pure utility functions (formatters, date helpers, aggregators)
  tests/         ← all test files as [feature-name].test.ts
```

### Separation of Concerns
- **components**: render only, receive props, emit events. No direct Supabase calls, no business logic
- **services**: all Supabase queries live here, all data transformation and aggregation logic
- **hooks**: compose services into stateful React logic, expose clean interfaces to components
- **context**: session management via React Context + localStorage for persistence
- **lib**: pure functions only — no side effects, no React, easily testable

### Cross-cutting Rules
- KISS: simplest solution that works
- DRY: extract shared logic, do not copy-paste
- SOLID: each file has one clear responsibility
- If a concern spans layers, define it in `lib/` or `services/` and import up

## Environment Variables
Injected by Doppler at runtime. Never hardcode. Never console.log.

```
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

The Supabase client is initialized once in `src/lib/supabaseClient.ts` and imported everywhere.
Do not re-initialize it elsewhere.

## Testing
- Every new feature gets a unit test
- Location: `src/tests/[feature-name].test.ts`
- Component tests: Vitest + React Testing Library
- Service/lib tests: Vitest only (pure functions, no DOM)
- No Supertest — there is no backend
- A feature is complete when: `npm run test` passes, `npm run build` succeeds, no lint errors

## Git Hygiene

### Commit Structure
```
[type/scope]: description
```

**Types:** `feat` `fix` `docs` `style` `refactor` `perf` `test` `build` `ci` `chore`

**Scopes:** `component` `service` `hook` `context` `lib` `test` `build` `ci`

**Description:** one line, max 50 chars, lowercase, imperative mood

### Examples
```
feat/service: add session aggregation by week
fix/hook: correct timer reset on project switch
test/lib: add unit tests for date formatter
```

### Commit Cadence
Commit at the end of each feature or sprint — not mid-feature unless explicitly asked.

## Commands

```bash
                 # doppler run -- npm run dev
npm run test     # run vitest

```

Always run via Doppler locally to ensure env vars are injected.

## Do Nots
- Do not add Claude or any AI tool as git co-author
- Do not hardcode or log env vars
- Do not call Supabase directly from components
- Do not add business logic to hooks — that belongs in services
- Do not create a new Supabase client instance — use `src/lib/supabaseClient.ts`
- Do not modify the DB schema directly — note required changes and flag for review
- Do not install new dependencies without flagging it first
